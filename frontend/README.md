# KB.AI Operations Console (Frontend)

Современный интерфейс управления платформой KB.AI на React + Vite + TailwindCSS.

## Что здесь есть

- панель мониторинга ingestion, поиска, чата и файлов;
- темная/светлая темы и локализация (RU/EN);
- загрузка документов drag-and-drop, предпросмотр и чат с LLM;
- админка с управлением пользователями, API-ключами и настройками;
- типизированные REST-клиенты и уведомления об ошибках.

## Требования

- Node.js 20+
- npm 9+

## Установка (как для 12-летнего)

1. Открой терминал и перейди в папку репозитория `kb_ai`.
2. Выполни команду:

   ```bash
   cd frontend
   npm install
   ```

   Это скачает все нужные библиотеки.

## Запуск dev-сервера

```bash
npm run dev
```

После запуска открой в браузере адрес <http://localhost:5173>. Страницы автоматически обновляются при изменении кода.

## Запуск линтера и форматтера

```bash
npm run lint
npm run format
```

Если `lint` ругается, исправь ошибки или запусти `npm run format:write` для автоисправления.

## Тесты

```bash
npm run test
```

Команда запускает Vitest с покрытием. Минимальное покрытие — 80% по строкам и функциям. Отчет `lcov` лежит в `coverage/`.

## Продакшн-сборка

```bash
npm run build
npm run preview
```

`build` собирает оптимизированную версию, `preview` проверяет её локально.

## Переменные окружения

Создай `.env` на основе `.env.example`:

```env
VITE_API_BASE_URL=http://localhost:8000/api/v1
VITE_WS_BASE_URL=ws://localhost:8000/ws
```

## Типичные проблемы

| Симптом | Что сделать |
| --- | --- |
| Браузер не подключается к API | Проверь `VITE_API_BASE_URL` и что backend запущен на `localhost:8000`. |
| WebSocket не открывается | Убедись, что `VITE_WS_BASE_URL` совпадает с настройками backend. |
| Ошибки `randomUUID` в тестах | Обнови Node.js до 18+ или добавь полифилл `global.crypto.randomUUID`. |
| Tailwind классы не применяются | Перезапусти `npm run dev`, Tailwind пересканирует файлы. |

Готово! Ты можешь править компоненты, добавлять страницы и сразу видеть результат в браузере.
