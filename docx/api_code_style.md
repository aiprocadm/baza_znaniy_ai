# API Code Style

Этот документ описывает практики, применяемые в HTTP-слое FastAPI. Правила
направлены на предсказуемость поведения и защиту от побочных эффектов.

## Иммутабельность входных моделей

* Никогда не мутируйте объекты запросов, полученные из Pydantic.
* Для корректировки данных используйте `model_copy(update=...)` (Pydantic v2) или
  создавайте отдельный DTO/доменные сущности.
* Обработчики должны работать с валидированными структурами. Если требуется
  изменить значение (например, нормализовать коэффициент), создайте копию и
  передайте её дальше по цепочке.

## Работа с файловыми загрузками

* Преобразуйте «сырые» объекты (`UploadFile`, словари, списки) в отдельные
  доменные объекты без изменения исходных инстансов. В `app/api/v1/upload.py`
  это реализовано через фабрику `create_upload_file`.
* Метаданные (`filename`, `content_type`) извлекаются и передаются в новые
  экземпляры, исходные объекты остаются неизменными.
* Очистка ресурсов должна учитывать как нормализованные, так и исходные
  объекты, чтобы гарантировать закрытие файловых дескрипторов.

## Общие требования

* Все входные данные проходят через Pydantic-схемы или явные валидаторы.
* Любые дополнительные проверки (границы чисел, формат строк) выносятся в
  доменные функции уровня сервиса.
* Исключения FastAPI создаются с явными кодами и короткими машиночитаемыми
  деталями (`detail="INVALID_SCALING"`).

Следуя этим правилам, мы избегаем нестабильных побочных эффектов и облегчаем
тестирование контроллеров.
