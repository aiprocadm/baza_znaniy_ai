# KB.AI — локальная база знаний с RAG

KB.AI — это одноконтейнерный сервис на FastAPI, который позволяет загружать
документы, автоматически индексировать их в векторном хранилище и получать
ответы чат-ассистента с цитатами. Репозиторий содержит минимальный набор, чтобы
за 15–20 минут развернуть окружение, обработать первый файл и получить
осмысленный ответ.

## Цели и основные возможности

- Веб-страница «Operations Console» для загрузки файлов, мониторинга индексации,
  поиска и чата на одной вкладке.
- Индексация документов (PDF/DOCX/TXT/XLSX/PPTX/MD) с разбиением на токены и
  сохранением чанков в Qdrant.
- Чат с локальной моделью через Ollama или со встроенной заглушкой, чтобы можно
  было тестировать без LLM.
- Хранение истории диалогов и промежуточных данных в каталоге `DATA_DIR`.

## Архитектура и компоненты

- **FastAPI + Uvicorn** — основное API (`app/api`) и статика (`app/ui/index.html`).
- **IngestService** — асинхронный сервис разбора файлов (`app/ingest`), который
  использует `tiktoken`, а при его отсутствии автоматически переключается на
  символьный токенизатор.
- **Vector store** — по умолчанию Qdrant (`VECTOR_BACKEND=qdrant`), но есть
  альтернатива `faiss` для автономного запуска (`app/retriever`).
- **LLM провайдер** — Ollama (`LLM_PROVIDER=ollama`), с fallback на
  детерминированный `stub`, который возвращает предсказуемые ответы в средах без модели
  (`app/llm/providers.py`).
- **Слой хранения** — SQLite для событий ingestion и истории чата в `DATA_DIR/db`
  (`app/core/deps.get_data_dir`).

## Требования

- Python 3.12+
- Docker (опционально, для контейнерного запуска)
- Git

## Настройка Python-окружения

```bash
python3.12 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

## Переменные окружения и `DATA_DIR`

1. Создайте `.env` из шаблона и скорректируйте параметры под свои сервисы:
   ```bash
   cp .env.example .env
   ```
2. Ключевые переменные:
   - `DATA_DIR=./var/data` — базовый каталог приложения. Создаётся автоматически
     при первом обращении и содержит подкаталоги `files/` и `db/` для загрузок и
     SQLite (`app/core/deps.py`).
   - `QDRANT_URL`, `QDRANT_API_KEY`, `QDRANT_COLLECTION` — параметры внешнего или
     локального Qdrant.
   - `LLM_PROVIDER`, `LLM_MODEL_NAME`, `OLLAMA_BASE_URL` — настройки модели.
   - `MAX_UPLOAD_MB`, `UPLOAD_ALLOWED_EXTS` — лимиты загрузки.
3. Если Ollama или Qdrant недоступны, установите `LLM_PROVIDER=stub` и
   `VECTOR_BACKEND=faiss`, чтобы использовать чисто локальные заглушки.

## Локальный запуск через Uvicorn

```bash
source .venv/bin/activate
uvicorn app.api.main:app --host 0.0.0.0 --port 8000 --reload
```

Конфигурация из `.env` автоматически подхватывается через `pydantic-settings`
(`app/core/config.py`). Статика доступна по `http://localhost:8000/`.

## Запуск в Docker

```bash
docker compose up --build
```

Compose-файл поднимает единственный сервис `kb_web`, монтирует `./var/data` в
`/app/var/data` и пробрасывает порт `8000`. Дополнительные nginx или внешние
volume'ы не требуются.

## Быстрый старт: путь к первому ответу

1. **Подготовьте окружение.** Выполните шаги по установке Python 3.12 либо
   поднимите контейнер.
2. **Создайте `.env`.** Убедитесь, что указали `QDRANT_URL` и `OLLAMA_BASE_URL`
   (или включили stubs).
3. **Запустите сервис.** Через `uvicorn` или `docker compose up --build`.
4. **Откройте консоль.** Перейдите на `http://localhost:8000/` — там единая
   страница управления загрузками, индексацией и чатом.
5. **Загрузите документ.** Используйте блок «Загрузка документа», дождитесь 100%
   прогресса и появления записи в таблице статусов.
6. **Проверьте индексацию.** Таблица «Статусы индексации» обновляется кнопкой
   «Обновить статус» или автоматически при успешной загрузке. В Docker можно
   смотреть `docker compose logs -f kb_web` для подробностей.
7. **Сделайте пробный поиск.** Введите запрос в блоке «Поиск по базе», чтобы
   убедиться, что чанки доступны.
8. **Начните чат.** Используйте форму «Чат с ассистентом»; ответы будут содержать
   цитаты на загруженные фрагменты.
9. **Траблшутинг.** Если чат не отвечает, проверьте доступность Ollama/Qdrant и
   значения `LLM_PROVIDER`/`VECTOR_BACKEND`. При отсутствии `tiktoken` сервис
   автоматически перейдёт на символьный токенизатор, что снижает качество, но не
   блокирует обработку.
10. **CI-проверка.** Перед пушем выполните `pytest` — тесты используют локальные
    stubs для внешних сервисов (`tests/service_stubs.py`), поэтому они проходят
    без Ollama и Qdrant.

## Дополнительные рекомендации

- Каталог `DATA_DIR` можно вынести на внешний диск, просто указав абсолютный
  путь в `.env`; приложение само создаст директории.
- При работе в stub-режиме ответы чата предсказуемы и подходят для smoke-тестов,
  но не для продуктивных сценариев.
- Для принудительной переиндексации удалите запись в `DATA_DIR/files` и загрузите
  документ заново — в хранилище будут созданы свежие чанки.
